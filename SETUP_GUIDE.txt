7. TROUBLESHOOTING
===============================================================

Problem: "Connection failed" error
Solution:
- Check if MySQL is running in XAMPP
- Verify database name in config.php
- Verify username/password in config.php
- Make sure database is imported

Problem: "Access Denied" for database
Solution:
- In phpMyAdmin, check user privileges
- Go to User Accounts ‚Üí root
- Make sure "localhost" has all privileges
- Or create new user with all privileges

Problem: Pages show blank/white screen
Solution:
- Enable error reporting temporarily
- Add to top of config.php:
  error_reporting(E_ALL);
  ini_set('display_errors', 1);
- Reload page to see actual error
- Fix the error
- Remove error reporting lines

Problem: Images not uploading
Solution:
- Check if uploads folder exists
- Check folder permissions (777)
- Check file size (max 5MB in post_item.php)
- Check file extension (only jpg, jpeg, png, gif)
- Add default.jpg in uploads folder

Problem: Login/Register not working
Solution:
- Check if users table has data
- Verify password is being hashed
- Clear browser cookies/cache
- Check session is started in config.php
- Verify form action points to correct file

Problem: Navigation links broken
Solution:
- Check BASE_URL in config.php
- Make sure it ends with /
- Verify folder name matches URL
- Check file names are correct (case-sensitive)

Problem: 404 Page Not Found
Solution:
- Verify folder structure is correct
- Check all files are uploaded
- Make sure .php extension is in filename
- Verify file paths in includes

Problem: CSS not loading
Solution:
- Check css/style.css file exists
- Verify BASE_URL in header.php
- Clear browser cache (Ctrl+F5)
- Check file permissions

Problem: Admin panel not accessible
Solution:
- Login with admin credentials
- Check user role in database (should be 'admin')
- Verify isAdmin() function in config.php
- Check admin folder exists
